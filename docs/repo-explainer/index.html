Research system · implementation audit · July 2026

Can MRI contrast teach meaning?

The project asks whether a vision transformer can learn small, semantically meaningful tissue representations by predicting how the same physical location should match across registered MRI modalities.

The outcome is methodology, not a leaderboard

BraTS/MET is the validation instrument. The desired object is a useful representation at a 4 mm physical footprint, while every crop entering the model is resized to 16 × 16 × 16 samples.

00 · The precise claim

One hypothesis, three meanings of small

Cross-modal hard matching may train a more semantic local representation than mean-seeking pixel reconstruction. That is a registered question, not a result. Three receptive fields must remain distinct.

implemented

4 mm carrier

The primary crop is 4 × 4 × 4 mm: only 64 native samples on the case-specific 1 mm grid before resizing.

implemented

96-token pretraining context

The student token begins from one crop but global attention mixes the entire sparse bag: three visible modalities × 32 positions.

different evaluation context

Four co-located evaluation tokens

The primary implemented downstream feature concatenates four jointly contextualized modalities at one location, without neighboring positions.

Do not conflate footprint with receptive field. The blind teacher sees exactly one crop. A pretraining student token can see a whole-case bag. The joint downstream view sees all four modalities but no spatial neighborhood.

02 · The implemented pretext task

Follow one hidden patch

Cross-modal hard matching does not reconstruct pixels. It selects, among eight candidate teacher embeddings from the hidden modality, the one belonging to the same registered position.

Largest conceptual caveat: the 32 sites are selected from a 512-center pool spanning the eligible case—not a compact neighborhood around a random anchor. Success can partly reflect aligned whole-brain anatomy and sparse relative geometry.

03 · Shortcut and leakage audit

Position is not one thing

Removing coordinates from the teacher blocks a trivial target code. It does not remove geometry from the student: source tokens and target queries receive relative rotary position information.

residual shortcut

Sparse whole-brain geometry

Distant samples form an anatomical constellation. Relative position plus approximate alignment may narrow the match before source appearance is understood.

04 · downstreamPathology structure

Richer MET labels and a lightweight dense readout.

Context shift: pretraining mixes 96 tokens across 32 positions with one hidden modality per site. The implemented primary evaluation supplies all four modalities at one location and no neighbors. The documented leave-one-modality-out sensitivity and lightweight dense segmentation readout are not built.

Collapse-monitoring scope: the fixed long-run probe is stable but uses four SSL-training subjects, so it is an optimization diagnostic rather than held-out representation evidence. Variance, effective rank, and off-diagonal cosine are implemented; the registered EMA-drift statistic is not.

Online W&B, local durability

  1. Verify literal detached LAUNCH_SHA.
  2. Use #SBATCH --propagate=NONE to clear login CPU limits.
  3. Run a separate compute-node W&B connectivity smoke.
  4. Run the sustained 160-step throughput/resume gate without network logging.
  5. Only then launch fresh scientific output.

Cadence: scalars and throughput telemetry every 10 steps; atomic local checkpoint every 1,000; one provenance-keyed artifact collection version every 5,000.

5k receipt: W&B upload is awaited, then an atomic local receipt binds its committed artifact identity to the checkpoint digest. A resume repairs any missing receipt before another batch or optimizer step.

W&B initialization and callbacks restore Python, NumPy, CPU Torch, and all CUDA RNG states. Canonical JSONL, plans, checkpoints, and reports remain local if transport fails.

Current truth: the online code path and connectivity smoke are implemented and locally tested. Live server connectivity has not been claimed because the cluster is down.

08 · Clean decision order

What happens next—and why

The sequence separates engineering readiness from scientific evidence, prioritizes falsification, and keeps soft targets out of the feedback loop until hard matching is understood.

  1. Finish small / 4 mm / hard matching

    Validate training, live tracking, checkpointing, and evaluation mechanics—not the hypothesis.

  2. Implement cross-modal reconstruction

    C versus D is the causal comparison that tests the mean-seeking motivation.

  3. Run signal-source controls

    No-coordinate, coordinate-only, content shuffle, co-located-only, context-only, fixed teacher.

  4. Freeze the evaluation endpoint

    Name joint utility, singleton/leave-one-out isolation, pathology, retrieval, and label-efficiency roles.

  5. Screen capacity and scale

    Three paired seeds, small versus base, then 4 versus 8 mm at equal exposure.

  6. Only then soften

    Audit ambiguity independently, ignore narrow ambiguous negatives, then consider soft positives.

  7. Confirm one winner

    Five paired seeds and a predeclared representation margin. Pretext score alone never advances an arm.

  8. Expand semantic validation

    Add physiologic enhancement negatives, MET tissue compartments, and a lightweight dense readout.
